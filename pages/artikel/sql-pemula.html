🎯 Apa itu SQL?

SQL (Structured Query Language) adalah bahasa standar untuk mengelola dan memanipulasi database relasional. Dengan SQL, kamu bisa membuat, membaca, mengupdate, dan menghapus data (CRUD operations).

📖 SELECT - Mengambil Data

Perintah SELECT digunakan untuk mengambil data dari database.

SELECT Semua Kolom

-- Mengambil semua data dari tabel users
SELECT * FROM users;

SELECT Kolom Tertentu

-- Mengambil nama dan email saja
SELECT name, email FROM users;

WHERE - Filtering Data

-- Mengambil user dengan id = 1
SELECT * FROM users WHERE id = 1;

-- Mengambil user dengan umur > 18
SELECT * FROM users WHERE age > 18;

-- Menggunakan AND
SELECT * FROM users WHERE age > 18 AND city = 'Jakarta';

ORDER BY - Mengurutkan Data

-- Urutkan berdasarkan nama (A-Z)
SELECT * FROM users ORDER BY name ASC;

-- Urutkan berdasarkan umur (terbesar ke terkecil)
SELECT * FROM users ORDER BY age DESC;

LIMIT - Membatasi Hasil

-- Ambil 10 user pertama
SELECT * FROM users LIMIT 10;

-- Ambil 5 user dengan umur tertua
SELECT * FROM users ORDER BY age DESC LIMIT 5;

➕ INSERT - Menambah Data

-- Menambah satu user
INSERT INTO users (name, email, age) 
VALUES ('John Doe', 'john@example.com', 25);

-- Menambah multiple users
INSERT INTO users (name, email, age) VALUES
('Jane Smith', 'jane@example.com', 23),
('Bob Johnson', 'bob@example.com', 30);

✏️ UPDATE - Mengubah Data

-- Update email user dengan id = 1
UPDATE users 
SET email = 'newemail@example.com' 
WHERE id = 1;

-- Update multiple kolom
UPDATE users 
SET name = 'John Smith', age = 26 
WHERE id = 1;
⚠️ Peringatan: Selalu gunakan WHERE clause saat UPDATE atau DELETE! Jika tidak, semua data akan terubah/terhapus.

🗑️ DELETE - Menghapus Data

-- Hapus user dengan id = 1
DELETE FROM users WHERE id = 1;

-- Hapus semua user dengan umur < 18
DELETE FROM users WHERE age < 18;

🔗 JOIN - Menggabungkan Tabel

JOIN digunakan untuk menggabungkan data dari dua atau lebih tabel berdasarkan relasi.

INNER JOIN

-- Menggabungkan tabel users dan orders
SELECT users.name, orders.product, orders.price
FROM users
INNER JOIN orders ON users.id = orders.user_id;

LEFT JOIN

-- Ambil semua users, termasuk yang belum punya order
SELECT users.name, orders.product
FROM users
LEFT JOIN orders ON users.id = orders.user_id;
💡 Tips Belajar SQL:
1. Praktik di SQLite atau MySQL
2. Gunakan platform online seperti SQLBolt atau W3Schools
3. Buat database sendiri untuk latihan
4. Pelajari konsep normalisasi database
5. Pahami perbedaan INNER JOIN, LEFT JOIN, RIGHT JOIN

Aggregate Functions

-- Hitung jumlah users
SELECT COUNT(*) FROM users;

-- Rata-rata umur
SELECT AVG(age) FROM users;

-- Umur tertua dan termuda
SELECT MAX(age), MIN(age) FROM users;

-- Total harga orders
SELECT SUM(price) FROM orders;

GROUP BY

-- Hitung jumlah user per kota
SELECT city, COUNT(*) as total
FROM users
GROUP BY city;

-- Total pembelian per user
SELECT user_id, SUM(price) as total_spent
FROM orders
GROUP BY user_id
ORDER BY total_spent DESC;

Kesimpulan: SQL adalah skill fundamental untuk backend developer. Kuasai CRUD operations dan JOIN untuk mengelola database dengan efektif. Praktik adalah kunci!
