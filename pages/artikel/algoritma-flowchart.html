🎯 Apa itu Algoritma?

Algoritma adalah urutan langkah-langkah logis untuk menyelesaikan suatu masalah. Algoritma adalah fondasi dari pemrograman - sebelum menulis kode, kita harus merancang algoritma terlebih dahulu.

Karakteristik Algoritma yang Baik

  • Input: Memiliki 0 atau lebih input
  • Output: Menghasilkan minimal 1 output
  • Definiteness: Setiap langkah jelas dan tidak ambigu
  • Finiteness: Harus berhenti setelah sejumlah langkah
  • Effectiveness: Setiap langkah harus bisa dikerjakan

Contoh Algoritma Sederhana

Algoritma Membuat Kopi:

  1. Siapkan cangkir
  2. Masukkan kopi bubuk ke cangkir
  3. Panaskan air hingga mendidih
  4. Tuang air panas ke cangkir
  5. Aduk hingga rata
  6. Kopi siap disajikan

Cara Menulis Algoritma

Ada 3 cara umum menulis algoritma:

1. Deskriptif (Natural Language)

Algoritma Mencari Bilangan Terbesar dari 3 Bilangan:
1. Mulai
2. Input bilangan A, B, C
3. Jika A > B dan A > C, maka A adalah terbesar
4. Jika B > A dan B > C, maka B adalah terbesar
5. Jika tidak, maka C adalah terbesar
6. Tampilkan bilangan terbesar
7. Selesai

2. Pseudocode

ALGORITHM FindMax
INPUT: a, b, c
OUTPUT: max

BEGIN
    IF a > b AND a > c THEN
        max = a
    ELSE IF b > a AND b > c THEN
        max = b
    ELSE
        max = c
    END IF
    
    PRINT max
END

3. Flowchart

Representasi visual menggunakan simbol-simbol standar.

📊 Apa itu Flowchart?

Flowchart adalah diagram yang merepresentasikan algoritma atau proses menggunakan simbol-simbol standar. Flowchart memudahkan pemahaman alur logika program.

Keuntungan Flowchart

  • Mudah dipahami secara visual
  • Membantu identifikasi error dalam logika
  • Dokumentasi yang baik
  • Memudahkan komunikasi antar developer
  • Membantu dalam debugging

🔷 Simbol Flowchart

Simbol Dasar

  • Oval/Terminator: Start/End
  • Parallelogram: Input/Output
  • Rectangle: Process/Instruction
  • Diamond: Decision/Condition
  • Arrow: Flow direction
  • Circle: Connector (untuk flowchart yang panjang)
💡 Tips Membuat Flowchart:
- Mulai dari atas ke bawah atau kiri ke kanan
- Gunakan simbol yang konsisten
- Beri label yang jelas pada setiap decision
- Hindari flowchart yang terlalu kompleks
- Gunakan connector untuk flowchart panjang

💡 Contoh Praktis

Contoh 1: Menentukan Bilangan Genap/Ganjil

Algoritma:

1. Mulai
2. Input bilangan n
3. Hitung sisa bagi n dengan 2
4. Jika sisa bagi = 0, maka n adalah genap
5. Jika tidak, maka n adalah ganjil
6. Tampilkan hasil
7. Selesai

Pseudocode:

ALGORITHM CheckEvenOdd
INPUT: n
OUTPUT: result

BEGIN
    IF n MOD 2 = 0 THEN
        result = "Genap"
    ELSE
        result = "Ganjil"
    END IF
    
    PRINT result
END

Implementasi (JavaScript):

function checkEvenOdd(n) {
    if (n % 2 === 0) {
        return "Genap";
    } else {
        return "Ganjil";
    }
}

console.log(checkEvenOdd(10)); // "Genap"
console.log(checkEvenOdd(7));  // "Ganjil"

Contoh 2: Menghitung Faktorial

Algoritma:

1. Mulai
2. Input bilangan n
3. Set faktorial = 1
4. Set i = 1
5. Selama i <= n, lakukan:
   - faktorial = faktorial * i
   - i = i + 1
6. Tampilkan faktorial
7. Selesai

Pseudocode:

ALGORITHM Factorial
INPUT: n
OUTPUT: result

BEGIN
    result = 1
    FOR i = 1 TO n DO
        result = result * i
    END FOR
    
    PRINT result
END

Implementasi (JavaScript):

function factorial(n) {
    let result = 1;
    for (let i = 1; i <= n; i++) {
        result *= i;
    }
    return result;
}

console.log(factorial(5)); // 120
console.log(factorial(3)); // 6

Contoh 3: Mencari Nilai Maksimum dalam Array

Algoritma:

1. Mulai
2. Input array angka
3. Set max = angka[0]
4. Untuk setiap angka dalam array:
   - Jika angka > max, maka max = angka
5. Tampilkan max
6. Selesai

Implementasi:

function findMax(arr) {
    let max = arr[0];
    for (let i = 1; i < arr.length; i++) {
        if (arr[i] > max) {
            max = arr[i];
        }
    }
    return max;
}

const numbers = [5, 2, 9, 1, 7];
console.log(findMax(numbers)); // 9

  • Draw.io (diagrams.net) - Free, web-based
  • Lucidchart - Professional tool
  • Microsoft Visio - Desktop application
  • Flowgorithm - Khusus untuk algoritma programming

✨ Kesimpulan

Algoritma dan flowchart adalah fondasi penting dalam pemrograman. Sebelum menulis kode, selalu rancang algoritma terlebih dahulu untuk memastikan logika program benar. Flowchart membantu visualisasi algoritma dan memudahkan komunikasi dengan tim. Latih kemampuan membuat algoritma dengan menyelesaikan berbagai problem solving!
