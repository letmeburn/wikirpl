🎯 Apa itu Git?

Git adalah version control system yang memungkinkan kamu melacak perubahan kode, berkolaborasi dengan tim, dan mengelola berbagai versi project.

⚙️ Instalasi Git

# Cek apakah Git sudah terinstall
git --version

# Konfigurasi user
git config --global user.name "Nama Kamu"
git config --global user.email "email@example.com"

📚 Perintah Dasar Git

1. Inisialisasi Repository

# Membuat repository baru
git init

# Clone repository yang sudah ada
git clone https://github.com/username/repo.git

2. Staging & Commit

# Cek status file
git status

# Tambahkan file ke staging area
git add filename.txt
git add .  # Tambahkan semua file

# Commit perubahan
git commit -m "Pesan commit yang jelas"

# Lihat history commit
git log
git log --oneline  # Format ringkas

3. Branching

# Lihat semua branch
git branch

# Buat branch baru
git branch feature-login

# Pindah ke branch
git checkout feature-login

# Buat dan pindah ke branch baru (shortcut)
git checkout -b feature-register

# Merge branch
git checkout main
git merge feature-login

# Hapus branch
git branch -d feature-login
💡 Best Practice Branching:
- main/master: Production code
- develop: Development code
- feature/*: Fitur baru
- bugfix/*: Perbaikan bug
- hotfix/*: Perbaikan urgent di production

🐙 GitHub Workflow

Push ke GitHub

# Tambahkan remote repository
git remote add origin https://github.com/username/repo.git

# Push ke GitHub
git push origin main

# Push branch baru
git push -u origin feature-login

Pull & Fetch

# Ambil perubahan terbaru
git pull origin main

# Fetch tanpa merge
git fetch origin

Pull Request Workflow

  1. Fork repository
  2. Clone fork ke local
  3. Buat branch baru untuk fitur
  4. Commit perubahan
  5. Push ke fork
  6. Buat Pull Request di GitHub
  7. Tunggu review dan merge

Mengatasi Conflict

# Saat terjadi conflict
# 1. Buka file yang conflict
# 2. Edit dan pilih versi yang benar
# 3. Hapus marker conflict (<<<<, ====, >>>>)
# 4. Add dan commit

git add .
git commit -m "Resolve conflict"
🎯 Tips Git:
✅ Commit sering dengan pesan yang jelas
✅ Pull sebelum push untuk menghindari conflict
✅ Gunakan .gitignore untuk file yang tidak perlu di-track
✅ Jangan commit file sensitive (password, API key)
✅ Review code sebelum merge

Perintah Berguna Lainnya

# Undo perubahan
git checkout -- filename.txt  # Undo file yang belum di-stage
git reset HEAD filename.txt   # Unstage file
git reset --hard HEAD~1       # Undo commit terakhir (HATI-HATI!)

# Stash (simpan perubahan sementara)
git stash
git stash pop

# Lihat perbedaan
git diff
git diff --staged

Kesimpulan: Git adalah tool wajib untuk developer modern. Kuasai basic commands dan workflow untuk kolaborasi yang efektif. Practice makes perfect!
