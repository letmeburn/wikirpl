NoSQL vs SQL: Memilih Database yang Tepat

🎯 Pengenalan

Memilih database yang tepat adalah keputusan penting dalam development. SQL dan NoSQL adalah dua paradigma database yang berbeda, masing-masing dengan kelebihan dan use case tersendiri.

🗄️ SQL Database (Relational)

SQL (Structured Query Language) databases adalah relational databases yang menyimpan data dalam tabel dengan schema yang fixed.

Karakteristik SQL

  • Schema Fixed: Struktur tabel harus didefinisikan sebelumnya
  • ACID Compliant: Atomicity, Consistency, Isolation, Durability
  • Relational: Data terhubung melalui foreign keys
  • Vertical Scaling: Scale up dengan hardware lebih powerful

Contoh SQL Databases

  • MySQL
  • PostgreSQL
  • Oracle
  • Microsoft SQL Server
  • SQLite

Contoh Query SQL

-- Create table
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Insert data
INSERT INTO users (name, email) 
VALUES ('John Doe', 'john@example.com');

-- Query with JOIN
SELECT u.name, o.order_date, o.total
FROM users u
INNER JOIN orders o ON u.id = o.user_id
WHERE u.id = 1;

-- Update
UPDATE users 
SET email = 'newemail@example.com' 
WHERE id = 1;

-- Delete
DELETE FROM users WHERE id = 1;

📦 NoSQL Database (Non-Relational)

NoSQL databases adalah non-relational databases yang lebih fleksibel dalam struktur data.

Tipe-tipe NoSQL

1. Document Store

Menyimpan data dalam dokumen (JSON, BSON, XML).

Contoh: MongoDB, CouchDB

// MongoDB - Insert document
db.users.insertOne({
    name: "John Doe",
    email: "john@example.com",
    age: 25,
    address: {
        city: "Jakarta",
        country: "Indonesia"
    },
    hobbies: ["coding", "gaming"]
});

// Query
db.users.find({ age: { $gt: 20 } });

// Update
db.users.updateOne(
    { email: "john@example.com" },
    { $set: { age: 26 } }
);

2. Key-Value Store

Menyimpan data sebagai key-value pairs. Sangat cepat untuk read/write sederhana.

Contoh: Redis, DynamoDB

// Redis
SET user:1:name "John Doe"
GET user:1:name
HSET user:1 name "John" email "john@example.com"
HGETALL user:1

3. Column-Family Store

Menyimpan data dalam column families. Cocok untuk big data.

Contoh: Cassandra, HBase

4. Graph Database

Menyimpan data sebagai nodes dan relationships. Cocok untuk social networks, recommendation engines.

Contoh: Neo4j, ArangoDB

Karakteristik NoSQL

  • Schema Flexible: Struktur data bisa berubah
  • BASE: Basically Available, Soft state, Eventually consistent
  • Horizontal Scaling: Scale out dengan menambah servers
  • High Performance: Optimized untuk specific use cases

⚖️ Perbandingan SQL vs NoSQL

Schema

SQL: Fixed schema, harus didefinisikan sebelumnya
NoSQL: Dynamic schema, fleksibel

Scalability

SQL: Vertical scaling (scale up)
NoSQL: Horizontal scaling (scale out)

ACID vs BASE

SQL: ACID - Strong consistency
NoSQL: BASE - Eventual consistency

Query Language

SQL: Standardized SQL
NoSQL: Varies by database (MongoDB query, Redis commands, etc.)

Relationships

SQL: Built-in dengan JOINs
NoSQL: Tidak ada JOINs, data sering denormalized

Performance

SQL: Bagus untuk complex queries
NoSQL: Sangat cepat untuk simple queries dan large volumes

💡 ACID vs BASE:
ACID: Atomicity, Consistency, Isolation, Durability - Garantee strong consistency
BASE: Basically Available, Soft state, Eventually consistent - Prioritize availability over consistency

🤔 Kapan Menggunakan Apa?

Gunakan SQL Jika:

  • Data terstruktur dengan relationships yang jelas
  • Butuh ACID compliance (banking, financial systems)
  • Complex queries dengan multiple JOINs
  • Data consistency sangat penting
  • Schema tidak sering berubah
  • Contoh: E-commerce, ERP, CRM systems

Gunakan NoSQL Jika:

  • Data tidak terstruktur atau semi-structured
  • Butuh horizontal scaling untuk big data
  • Schema sering berubah (rapid development)
  • High throughput untuk read/write operations
  • Eventual consistency acceptable
  • Contoh: Social media, IoT, real-time analytics, caching

Hybrid Approach

Banyak aplikasi modern menggunakan keduanya:

  • SQL untuk transactional data
  • NoSQL untuk caching (Redis)
  • NoSQL untuk analytics (MongoDB)
  • NoSQL untuk session storage

📊 Contoh Use Cases

SQL Use Cases

-- Banking: Transfer uang (ACID penting)
BEGIN TRANSACTION;
UPDATE accounts SET balance = balance - 100 WHERE id = 1;
UPDATE accounts SET balance = balance + 100 WHERE id = 2;
COMMIT;

-- E-commerce: Order dengan multiple items
SELECT o.id, o.total, u.name, p.name as product
FROM orders o
JOIN users u ON o.user_id = u.id
JOIN order_items oi ON o.id = oi.order_id
JOIN products p ON oi.product_id = p.id
WHERE o.status = 'pending';

NoSQL Use Cases

// Social Media: User profile dengan nested data
{
    "_id": "user123",
    "username": "johndoe",
    "profile": {
        "bio": "Software Developer",
        "avatar": "url",
        "location": "Jakarta"
    },
    "posts": [
        { "id": 1, "content": "Hello World", "likes": 100 },
        { "id": 2, "content": "Learning NoSQL", "likes": 50 }
    ],
    "followers": ["user456", "user789"]
}

// IoT: Sensor data (high write throughput)
db.sensors.insertMany([
    { sensor_id: "temp01", value: 25.5, timestamp: new Date() },
    { sensor_id: "temp02", value: 26.1, timestamp: new Date() }
]);

✨ Kesimpulan

SQL dan NoSQL bukan kompetitor, tapi tools yang berbeda untuk kebutuhan berbeda. SQL cocok untuk data terstruktur dengan relationships kompleks dan butuh strong consistency. NoSQL cocok untuk data fleksibel, high scalability, dan rapid development. Pahami kebutuhan aplikasi Anda dan pilih database yang paling sesuai - atau gunakan kombinasi keduanya!
